FAQ

MongoDB Atlas is a fully managed cloud database service that simplifies deploying, scaling, and managing MongoDB databases in the cloud. It eliminates infrastructure setup and database management by automating critical tasks such as provisioning, backups, scaling, and performance optimization. Atlas database supports multi-cloud deployments on AWS, Azure, and Google Cloud. This enables global scalability and resilience. Atlas is designed to ensure engineering teams focus on building and innovating applications rather than managing database infrastructure.

Learn more

MongoDB Atlas database uses a flexible pay-as-you-go pricing model. This allows organizations to only pay for the resources they use. It offers three cluster tiers:

  • Free Tier: Ideal for testing and exploring use cases, with no cost to get started.
  • Flex Tier: Pay for what you use with granular billing for storage, compute, and IOPS, perfect for unpredictable workloads.
  • Dedicated Tier: Fixed pricing for predictable, production-ready workloads.

Auto-scaling ensures cost efficiency by dynamically adjusting resources based on workload demands, preventing overprovisioning or underutilization.

Learn more

MongoDB Atlas delivers enterprise-grade security out of the box. This ensures sensitive data is protected at every stage:

  • Access control: Flexible role-based access controls and streamlined authentication ensure that only authorized users and applications can access data.
  • Encryption: Data is encrypted at rest, in transit, and in use to prevent unauthorized access, with no cryptography expertise required.
  • Automatic patches: Security patches are applied automatically, protecting deployments without operational disruption.
  • Auditing: Granular auditing provides visibility into database activity to meet compliance and detect security issues.
  • Regulatory compliance: Atlas supports multi-region clusters and is certified with over 15 compliance standards, helping meet data residency and privacy regulations for the most stringent compliance requirements.

These features make Atlas a reliable choice for enterprises with stringent security and compliance needs.

Learn more

Migrating to MongoDB Atlas database is straightforward, with tools designed to simplify the process:

  • Mongodump/mongorestore (compatible with all MongoDB versions): Used for legacy system migrations, cross-cloud transfers (e.g., AWS to Azure), and large dataset (>1TB) migrations.
  • Mongoimport/mongoexport (compatibility: both tools need to be on the same version e.g. v100.9.0): Best for small datasets (<100GB) due to JSON/CSV conversion overhead.
  • Atlas live migration service (compatible with Atlas v6.0.17+ or 7.0.13+): Enables seamless, live data migration from self-managed MongoDB without downtime.
  • Relational Migrator (compatible with all Atlas versions): Use this free tool to migrate from legacy relational databases to MongoDB Atlas. It features intelligent algorithms and gen AI to automate the most complex aspects of migration including schema modeling, updating application code, and converting SQL queries, views, and stored procedures.
  • Cluster-to-Cluster Sync (compatible with MongoDB 7.0+): Real-time hybrid cloud sync using Change Streams API. Supports filtered synchronization and 1:1 namespace mapping. Commonly used for disaster recovery and analytics mirroring.

Atlas minimizes disruptions, allowing developers and engineering leaders to leverage its fully managed capabilities quickly.

Learn more

MongoDB Atlas database is designed for resilience. This ensures high availability and enables robust disaster recovery through:

  • Distributed, self-healing deployments: Every deployment is a distributed three-node replica set. Automatic failover mechanisms ensure seamless continuity, including cross-region and cross-cloud failovers.
  • Continuous backups: Automated backups with point-in-time recovery support stringent recovery point and time objectives (RPO and RTO).
  • Multi-region, multi-cloud clusters: Data can be replicated across multiple regions and multiple cloud providers. This ensures resilience and compliance with data residency regulations.
  • 99.995% Uptime SLA: Atlas guarantees high availability for production workloads with automated failovers.
